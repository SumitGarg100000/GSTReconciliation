1. Introduction to GST Reconciliation Tool

The GST Reconciliation Tool developed by Sumit Garg is a specialized solution designed to streamline the process of reconciling GST data, making tax compliance simpler and more efficient for businesses and tax professionals in India.

What is GST Reconciliation?

GST reconciliation is the process of comparing and matching data in your books with the data in GST returns and the GST portal. This ensures compliance, accurate tax filing, and helps in claiming the correct Input Tax Credit (ITC) amounts.

This tool specifically addresses the challenges in matching data between GSTR-2B (data from suppliers' GSTR-1) and your books/GSTR-3B to ensure ITC claims are accurate and defensible during audit or scrutiny.

Key Benefits:

Speed

Reconciles data within seconds, saving hours of manual work

Accuracy

Eliminates human error in matching invoices and tax amounts

Flexibility

Works with various formats and tolerates minor data discrepancies

Security

Data remains secure as it's processed locally without cloud storage

The tool bridges the gap between complex GST data requirements and the practical needs of businesses, ensuring regulatory compliance while minimizing the time and effort involved in reconciliation processes.

4. Key Features

The GST Reconciliation Tool offers several powerful features that make it stand out from other reconciliation solutions:

Detailed Reconciliation

The tool provides a comprehensive reconciliation view that categorizes matches based on different criteria, giving users deeper insights into their data.

Feature Highlight:

Users can toggle between simplified and detailed reconciliation views. The detailed view breaks down matches into specific categories based on matching parameters.

The detailed reconciliation categorizes matches into seven distinct headings:

1. Match: GST No, Invoice No, Date

Perfect matches on all three key parameters

2. Match: GST No, Invoice No

Matches by GST and invoice numbers, ignoring date differences

3. Match: GST No, Date

Matches by GST number and date, useful for invoice number variations

4. Match: GST No Only

Matches by supplier GST number only (party-wise matching)

5. Unmatch: GST No Not Exist

GST numbers appearing in one dataset but not the other

6. Unmatch: Amount Difference

Invoices with matching identifiers but different amounts

7. Unmatch

Entries that don't match any criteria

This detailed classification helps users pinpoint exactly where discrepancies exist and why, making it easier to address specific issues.

Filtering Options:

To use filtering, click the "Enable Filter" button, and filter icons will appear in each column header. Click on these icons to set specific filtering criteria.

Sorting Capabilities:

A-Z Sorting

Sort data in ascending order (alphabetical or numerical)

These filtering and sorting capabilities eliminate the need to export data to Excel for analysis, as all necessary data manipulation can be performed directly within the tool's interface.

Match Criteria Explanation

The tool employs sophisticated matching logic that goes beyond simple exact matching to account for common variations in data entry across different systems.

Intelligent Matching Features:

Character Insensitivity

Matches uppercase and lowercase text (e.g., "INV001" matches "inv001")

Special Character Handling

Ignores spaces, dots, slashes, and special characters in invoice numbers

Multi-level Matching

Progressively relaxes match criteria if exact matches aren't found

Party-wise Aggregation

Can match at the supplier level when invoice details vary

Example:

If one system records an invoice as "INV/2025-26/001" and another as "INV202526001", the tool will still recognize these as the same invoice by ignoring the special characters and focusing on the significant parts of the invoice number.

This intelligent matching significantly reduces false negatives in reconciliation, preventing the need for manual intervention in cases where the data is essentially the same but formatted differently.

Difference Allowed Feature

The "Difference Allowed" feature is a powerful capability that lets users set tolerance levels for amount differences, accommodating minor variations that often occur in real-world accounting scenarios.

How It Works:

Users can specify a rupee amount as the acceptable difference threshold. If the difference between matched entries falls within this threshold, the entries will be considered matched despite the variation.

Practical Application:

This feature is particularly useful for handling rounding differences, minor calculation variations, or small discrepancies that are not material enough to warrant investigation or correction.

Setting the Difference Threshold:

  1. Locate the "Difference Allowed" field in the tool interface
  2. Enter the maximum acceptable difference amount (e.g., ₹5)
  3. The system will automatically apply this tolerance to all comparisons

By default, the system sets a ₹1 difference threshold, which can be adjusted according to specific reconciliation needs and organizational policies.

Example Scenario:

If a supplier invoice shows ₹9,000 tax value while your books record ₹9,002:

  • With ₹1 difference allowed: These would be classified as "Unmatch: Amount Difference"
  • With ₹5 difference allowed: These would be considered matched since ₹2 is within the tolerance
